# 六.多用组合少用继承

### 1.通过组合加委托来替换继承关系，解决继承劣势，消除代码的重复定义

### 2.如果类之间的继承结构稳定（不会轻易改变），继承层次比较浅（比如，最多有两层继承关系），继承关系不复杂，我们就可以大胆地使用继承。反之，系统越不稳定，继承层次很深，继承关系复杂，我们就尽量使用组合来替代继承。
